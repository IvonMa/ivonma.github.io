WebService的使用

发表于 2017-08-08 | 分类于 后端 | |

一种跨编程语言和跨操作系统平台的远程调用技术规范。用来实现不同语言或系统之间的数据交互。

cxf 是WebService在Java语言中的一种实现框架。

按固定的格式来传递信息。

三规范

Java中共有三种WebService规范：

  1. JAX-WS(JAX-RPC)

    全称 Java API for XML-Based WebService

    • 开发简单，高度封装实现细节
    • 比较笨重，需要导入大量 Jar
    • 目前已经被 JAX-RS 取代，大多是传统项目在用

  2. JAXM&SAAJ

    Java API For XML Message

    • 暴露了更多的SOAP底层细节，编码比较麻烦

  3. JAX-RS

    Java针对REST(Representation State Transfer) 风格制定的一套Web服务规范。

    • 比较新，08年才有了具体的实现，随JDK1.7发布
    • 目前电商系统更多采用

三要素

1.SOAP

soap协议是WebService的传输协议，即简单对象访问协议。
Soap协议是xml格式，基于http传输xml数据。

特点：跨平台，跨语言，w3c指定

2.WSDL

WebService 的使用说明书。
Web Service Description Language 网络服务描述语言，是一种使用XML编写的文档，用于描述或定位网络服务。

例如，其中描述了接口提供的方法的方法名，参数和返回值等

3.UDDI

统一描述、发现和集成协议
是一个目录服务，存储了全球的WebService地址。

CXF 框架

简介

什么是CXF

Celtix + Xfire
是一个Java下的开源WebService框架。
支持多种协议，比如：SOAP1.1, 1.2, XML/HTTP, REST HTTP, CORBA
以下使用的是JAX-WS，但CXF也支持JAX-RS。

服务器端开发步骤：

  1. 导入Jar包

  2. 创建服务类及接口，添加供远程调用的方法

    • 接口必须有@WebService注解
    • 实现类的方法必须有@WebMethod注解

  3. 发布WebService：

    非Spring整合方式发布

  4. 创建生产服务的工厂类

    JaxWsServerFactoryBean factoryBean = newJaxWsServerFactoryBean ();

  5. 设置访问地址（提供给外界访问本服务的地址）

    factoryBean.setAddress("http://localhost:12345/weather"");

  6. 设置具体服务对象

    factoryBean.setServiceBean(new WeatherService());

  7. 创建服务

    factoryBean.create();

Spring整合方式发布：

  1. Spring配置文件中配置

    • address属性定义的是服务的路径，即?wsdl前面那个路径。注意其与项目路径和CXFServlet的url-pattern的区别：

      假设项目路径为ivon，web.xml中CXFServlet的url-pattern为ws，此处address为/weather，则最后访问wsdl的路径为：http://localhost:8081/ivon/ws/weather?wsdl

  2. web.xml 配置 CXF的核心控制器CXFServlet

    • 特别注意url-pattern最后要加通配符： /xx/* 这是由于客户端对其的访问必然会在Servlet的url后拼接？？？
