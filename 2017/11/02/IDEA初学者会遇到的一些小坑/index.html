IDEA初学者会遇到的一些小坑

发表于 2017-11-02 | 分类于 工具 | |

IDEA虽好，但学习成本不低。并且，由于它与Eclipse在项目创建、发布等方面差异巨大，常常会令初学者浅尝辄止。本文总结了我本人在学习和使用过程中遇到过的一些小坑，希望对大家有所帮助。

  • IDEA默认是不会热部署任何页面的，包括但不限于html、jsp、js、css。
  • Build/Rebuild 项目或模块,在Project面板里不一定是立即显示效果的.例如classes目录里的文件,常常20+秒才出现.但如果用资源管理器打开来看到底好没好,可以立即看到效果.
  • 服务器运行状态Build/Rebuild 项目或模块，常常会出问题。例如配置文件不进classpath之类的.
  • 只有添加了Web Application,才能在新建里选择Servlet/Listener等模版

  • 问题:Struts核心过滤器org.apache.struts2.dispatcher.ng.filter.StrutsPrepareAndExecuteFilter报红

    解决：idea需要手动添加Tomcat的包,或JavaEE包

  • 问题:网页文件输出位置异常.

    解决：要么是Facets-Web-WebResourceDirectories下面配置的有问题,要么是Artifacts-OutputDirectory错了

  • resource配置文件目录下创建子目录时，必须使用 cn/itcast这样的形式。如果用点，如cn.itcast，则只会创建一个cn.itcast为名字的文件夹，并不会有分级目录。并且，两者显示的是一模一样的，真假难辨。

  • 问题：忽然无法更新JSP页面上的内容了。无论怎么修改，重启IDEA的TOMCAT服务器后都见不到效果。

    解决方案：看看项目的web设置中的路径是不是变成默认了，即因为跟现在路径不符而变红了

  • 问题：properties文件在其他配置文件中的引入，无法通过cmd+左键导航到properties中的对应值。引入没有任何问题，且一般项目运行起来也没有问题。

    解决：项目设置中没有将引入了properties的该配置文件设置为其对应框架的配置文件。

  • 任何请款下，需要删除Web项目输出位置的web文件时，一定要将整个输出目录都删掉（例如target）。倘若只是删掉个别文件，那么无论是重启服务器还是重编译项目，都不会将该Web文件重新输出。
