基于Hexo与GitHub的建站笔记

发表于 2017-11-26 | 分类于 其他 | |

为何使用Hexo

最常用的静态网站模板引擎显然是Jekyll，它有着目前最大的模板生态，且被GitHub等提供Page服务的代码托管网站直接支持，只需要push Jekyll项目本身即可由Page服务自动转为静态网站发布。

  • 对于GitHub而言：上传到项目的gh-pages分支即可直接解析为静态页面
  • 对于码云而言：需要在项目主页的 服务=>Pages 中选择开启Page静态网站服务。可以指定分支。

与Jekyll不同，Hexo在本地生成静态网站，而后发布到Git远程仓库上去的是已经生成好的静态网站，不需要Page服务来做转换。这样做的好处是，可以绕过GitHub Page为了安全性考虑而禁用jekyll插件的问题，在生成静态网站时获得更强的拓展性；且由于不需要提交项目本身到公共仓库，可以更好地保护可能存储在配置文件中的私有信息，如各种api Key。

虽然jekyll也可以在本地生成静态网站而后直接提交，但hexo对此的支持更好。例如它提供了基于Node.js的本地服务器用于本地预览生成的静态页面，再例如使用它的hexo deploy命令可以直接生成静态网站并提交到远程Git仓库。

创建项目

常用命令

  • 清理本地已生成的网页文件：hexo clean
  • 生成网页文件：hexo generate
  • 本地运行测试服务器：hexo s -d
  • 发布到远程Git仓库：hexo deploy
    • 需要先在站点配置文件中添加deploy信息。
    • 错误提示：ERROR Deployer not found: git，解决方式：npm install --save hexo-deployer-git。注意，插件是为当前目录下hexo项目安装，而不是全局。

评论系统：采用Gitment

简介

Gitment是一个基于GitHub仓库的Issue系统的评论系统。

原理

对每个页面的评论，实际上会被作为该仓库的一条Issue存在。只要页面绝对地址不变，那么即使曾经删除页面，恢复后依然可以对应上原有评论。

缺点

只支持GitHub用户评论。

步骤

  1. 点击此处 去GitHub来注册一个新的 OAuth Application
  2. 在主题配置文件中将gitment的enable设置为true，并根据上一步获取到的信息填写所有必填项即可。

注意

  • 5.13+版本Next主题已经支持直接集成Gitment系统。只需要在配置文件中打开即可，不需要手动引入JS。

参考：添加Gitment评论系统到Hexo主题NexT Gitment：使用 GitHub Issues 搭建评论系统

统计系统：采用不蒜子

  • Next主题已经集成了不蒜子统计，在主题配置文件中将busuanzi_count的enanble设置为true即可。其他可选配置项参见Next中文文档。

设置访问地址

  • 对于GitHub而言
    • 项目主页的默认访问地址是 https://用户名.github.io/项目名
    • 如果希望直接使用https://用户名.github.io/访问，只需将项目名设置为用户名.github.io
    • 地址中及项目名中的用户名都是小写
  • 对于码云而言
    • 项目主页的默认访问地址是 http://用户名.gitee.io/项目名
    • 与GitHub不同，如果希望直接使用https://用户名.gitee.io/访问，需将项目名设置为用户名
    • 地址中及项目名中的用户名都是小写

绑定域名

  • 对于GitHub而言
    • 项目主页=>Settings=>GitHub Page=>Custom domain 设置为买来的域名
    • 保证每次上传后远程仓库的根目录下都有CNAME文件，内容为自定义域名。
      • 对hexo而言，放到source目录下即可保证每次生成和上传后根目录下都有该文件。
  • 对于码云而言
    • 不支持自定义域名
